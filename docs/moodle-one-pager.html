What this program is

A hands-on, 10–12 week internship capstone where each trainee independently builds ProjectFlow — a full-featured, Kanban-style project management application similar to Trello or Asana. Every intern forks the base repository and develops a complete implementation on their own, covering authentication, database design, interactive UI, testing, and production deployment.

The program anchors to real engineering practices: typed schemas, server actions, CI/CD pipelines, and end-to-end test coverage. A mentor is available throughout and conducts milestone reviews. The final week ends with a live showcase where each intern presents their individual version of the same product — making architectural trade-offs visible and sparking meaningful technical discussion.

How you're assessed

Mentor milestone reviews (50%) — Four formal reviews at Phases 3, 5, 7, and 8. Each review is completed within 3 working days of submission and includes written feedback plus a 30-minute video walkthrough.

Live capstone showcase (30%) — A 15-minute live demo of the fully deployed application, followed by a technical Q&A. Scored on feature completeness, code quality, and ability to explain architectural decisions.

Code quality & practices (20%) — Evaluated across the full 12 weeks: commit discipline, TypeScript strictness, test coverage, documentation quality, and adherence to the feature-branch workflow. Minimum passing score: 3.0/5.0.

Earn your Full-Stack Next.js Capstone Badge

Pass all milestone reviews and the live showcase to earn your Stratpoint Full-Stack Next.js Badge, a certificate of completion, and a portfolio-ready deployed application for your GitHub profile.

Why individual builds?

Each intern builds the complete application — not a slice of it. You own every layer: auth, database, UI, tests, and deployment. This produces a genuine portfolio project and exposes you to architectural decisions you'd otherwise never face. Daily standups and optional code-review sessions keep the group connected despite working solo.

The one rule

Write the code yourself. AI assistants and copy-pasting are tempting shortcuts — they will also leave you unable to explain your own codebase in the live showcase or on the job. Struggle is where the learning lives. Use AI to understand, not to generate.
